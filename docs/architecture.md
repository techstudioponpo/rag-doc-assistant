# Architecture（システム概要設計）

## 目的

本プロジェクトは、ドキュメントをもとに質問応答を行うRAGアプリケーションの最小構成を実装し、
将来的な拡張を前提とした基盤を構築することを目的とする。

---

## 設計方針

### 1. シンプルな構成

* フロントエンドとバックエンドを分離
* 必要最小限の機能から実装
* 学習・拡張しやすい構成

---

### 2. RAGの基本構成を明確にする

本システムは以下の流れをコアとする。

1. 文書を分割（チャンク化）
2. Embeddingを生成
3. ベクトルデータとして保存
4. 類似検索
5. AIによる回答生成

---

### 3. 入出力の分離

* 入力（ドキュメント）と出力（回答）を分離
* 将来的に以下の差し替えを可能にする

  * PDF → Webページ / Notion / DB
  * UI → Web / API / チャットツール

---

### 4. 再現性の確保

* Docker Compose により環境を統一
* 誰でも同じ手順で起動可能にする

---

### 5. セキュリティ設計

* APIキーは環境変数で管理
* フロントエンドに機密情報を持たせない

---

## データ設計（概要）

### documents

* id
* title
* source_type
* created_at

### chunks

* id
* document_id
* chunk_index
* text
* embedding

---

## 拡張性

本設計により、以下のような拡張が可能。

* 複数ドキュメント対応
* カテゴリ分類
* 検索対象の絞り込み
* ユーザー単位のデータ管理

---

## まとめ

本プロジェクトは単なるPDF検索ツールではなく、
「検索 + AI」を組み合わせたRAG基盤として設計している。

小さく実装し、用途に応じて拡張可能な構造とする。
